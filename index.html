Floating with CSS

The CSS float property lets an element move to the left or right side of its container while nearby text wraps around it. Floats were once used for entire page layouts, but modern web designers usually use Flexbox and Grid for major layout work.

Today, floats are still useful for smaller layout effects, especially when you want text to wrap around an image, figure, pull quote, or other small piece of content.

In this exercise, we will use a photo of Ivan and Cookie to see how floating works. First, we will try floating the image by itself. Then we will wrap the image in a figure so the caption can move with the image as one connected unit.

This is an important idea in web design: sometimes you should style the individual element, and sometimes you should style the container that holds several related elements together.

As you work through the exercise, pay attention to how the text reacts when the image or figure is floated. The text does not simply move below the image. Instead, it flows around the floated element, almost like water flowing around a rock in a stream.

The second image gives us another chance to practice floating. Once you understand how the first floated figure works, we can use this image to try floating content in the opposite direction.

This also helps demonstrate why we do not usually want to float every image on the page at once. A broad rule like img { float: right; } affects every image, even when we only meant to target one specific image or figure.

After both figures have been floated, the next section may try to wrap around them. That is where the clear property becomes useful. Clearing a section tells the browser that the section should start below the floated content instead of squeezing beside it.

How Floats Work

The most common float values are left, right, and none. A left float moves the element to the left side of its container. A right float moves it to the right side. The value none removes the float and returns the element to normal document flow.

Floats can be useful, but they also have quirks. When an element is floated, surrounding text may wrap beside it. Sometimes, that's exactly what you want. Other times, later content may wrap in ways you didn't expect.

The clear property helps control what happens after a floated element. For example, clear: both; tells an element to move below any floated elements on either side. This is useful when you want a new section to start underneath a float instead of continuing beside it.

Floated images also need spacing. If there is no margin, the text may sit too close to the image. Adding margin on the side of the float creates breathing room and makes the page easier to read.

Finally, images often need a responsive image rule. A common rule is max-width: 100%; and height: auto;. This allows images to shrink to fit inside their containers while keeping their proportions correct.
